解法二：

目前看到了个格的解法(来自无趣的浅，又是这位大哥)，比较像预期解。

对于题目的分析:

$p1 = a b c + 1$

$p2 = b c d + 1$

$q1 = e d f + 1$

$p3 = a e f + 1$

$q2 = k1 e f + 1$

$q3 = k1 b c + 1$

$n1 = p1 * q1$

$n2 = p2 * q2$

$n3 = p3 * q3$

$ED = (a b c d e f) * k1 + 1$

进一步展开$n1,n2,n3$有：

$n1=abcdef+(abc+ed*f+1)$

$n2=bcdefk1+(bcd+k1e*f+1)$

$n3=abcefk1+(aef+k1b*c+1)$

再进一步构造：

$ED-k1n1=1-k1(abc+ed*f+1)$

$ED-an2=1-a(bcd+k1e*f+1)$

$ED-dn3=1-d(aef+k1b*c+1)$

记上3式等号右结果为A,B,C，即：

$ED-k1n1=A$

$ED-an2=B$

$ED-dn3=C$

分析各个数的大小有：

$E$-1533bit; $n1,n2,n3$-1534bit;$a,d,k1$-528bit;$D$-528bit;

$A，B，C$-1296(528+768)bit；

实操跑了一下t取$2^{600}-2^{800}$都可以，在此情况下，相对于1534位的格基，基本在1296位的向量确实是格上最短。

t下界的相关证明（队友问了L的大哥，说是用高斯启发式）：

忘记咋求行列式了，唐突补习：

$det L=tn1n2n3$ 期望最短向量长度为$0.48(det L)^{1/4}$,转换为比特位即：

$(1534*3+tbit)/4 \ =1150+0.25tbit \ 位$

|w|（目标向量长度）$>1296 \ 位$

可得若目标向量为格上最短向量，则tbit约$>584$ 位

实测为589，差别不大。

分析：

调整dbits = 0x240，使得原本easyRSA的脚本无论如何调整t的大小都无法获得结果。

同样的等式

$ED-k1n1=A$

各个数的大小有

$E$-1534bit; $n1,n2,n3$-1534bit;$a,d,k1$-576bit;$D$-576bit;

$A，B，C$-1344(576+768)bit；

复现的时候对于卡了t的界的问题自己提了个比较粗浅地推导：

对于easyRSA里的这个格，同样利用高斯启发式考虑。

$det L=tn1n2n3$ 期望最短向量长度为$0.48(det L)^{1/4}$,转换为比特位即：

$(1534*3+tbit)/4 \ =1150+0.25tbit \ 位$

|w|$>1344 \ 位$

为满足需求有$tbit>776 位$

但此时再次计算目标向量中的D*t，可以发现其位数为$576+776=1352>1344$，即此时目标向量长度改变，tbit不再能满足需求，也就推出矛盾了。

时间线稍后，队友复盘提到了格攻击之小未知数方程求解入门——原理与例子 | Tover’ Blog，里面提到了配平技巧。

文章的大致意思指，一般情况下先配平，再去考虑高斯启发式，否则格可能会无法满足需求，配平即使目标向量中各个数的大小平衡。

对于midRSA这个题而言，就是令tbit=768，有|w|=1344

此时可求得α(L)=1342.5<1344不满足要求，这样就可以看到所谓的卡界了。
